軟體說明

利用 venv 建立虛擬環境，並在環境中安裝 python PyQt5、 numpy，下 python main.py 的指令即可執行，或是直接下載 main.exe 直接執行即可使用





程式設計

資料結構

此 Voronoi 程式中最主要用到的資料結構就是 class 的 Edge，其中有紀錄兩點所構成的 edge 長度、 edge 的向量、 edge 的斜率、 edge 的中間點、 edge 的中垂線，以及構成 edge 的兩個點，最後再將所有的邊(class Egde)放入陣列當中

特別技巧

1. 雖然畫布只有600 * 600的大小，但是要把它視為超過600 * 600的大小，因為有可能外心在畫布的外面，如果只有限制在0～600的話就GG
2. 判斷是否為銳角三角形不需要去計算角度，只要去計算如果兩條較小的邊平方和不大於第三邊平方就代表是銳角

改良之處

1. 算出兩線或三線交會的那個點時，要對應到一條線的中間點，如果是垂直的話會有些許的誤差大概落在小數點後第一位，而這時我會給他一個區間為誤差的區間，因此如果計算兩點非常相近時，就會自動落入那個區間，則忽略掉，因此這部分需要改良
2. Merge 時的剪線也有出了問題，在4～6的點大多都正確，但超過6個點以後就要看天了，因此剪線的部分也需要改良

結論與心得

結論

首先在選擇程式語言上先選擇了Python，因為聽學長們推薦使用Python Qt5的套件，而在期中demo所做的是最基本的情況(三個點以下)，因為一開始是自己憑空想像，因此是沒有使用向量的概念，而是單純算邊、點都不含方向的概念，等到之後要切線時才發現自己無法決定要切哪一段的線，之後就要開始考慮很多的特殊情況，例如共線、平行、垂直。

期末的部分則是最大的魔王，一開始完全沒有想法，所以只好先寫convex hull，再來到找hyperplane時，完全不知道演算法該怎麼做，之後重複看老師在網路上的影片才有個頭緒，按照從上找到下，一個一個的中垂線去找對應到的兩個點，而這時候我的資料結構就出來問題，沒有記到我要使用到的資訊，之後才趕快將之前寫的資料結構再加上那個資訊，但是也就導致程式碼沒有很完整的考慮過再去實作，可能處處都會有bug的感覺，然後寫完hyperplane時發現又有bug，debug好一陣子才突然想到老師說hyperplane只會由上往下走，我就將這個條件設下，於是乎bug就找到了，最後在剪線的時候，詢問了許多學長，使用了很多種不同的方法，但是都還是會有bug，只好放棄7點以上，只拿6點以下的分數，因為我知道如果要找這個bug可能會需要動到更多之前的程式碼，因此我也懶了，所以最後就這樣寫了個半成品出來

心得

這次程式真的讓我感覺我的程式又更進步了，主要進步的點是對於一個抽象的概念能夠用程式一步一步實踐出來，而且對於一個大的目標可以切成很多小的目標，再一步一步慢慢地實踐出來，不管是對於程式能力還是在思考問題的想法都有明顯進步的感覺

可改進部分

希望一開始的時候就該去看老師提供的那個資料結構來實作，到後面再做 Merge 的時候需要一些資訊實才有完整且正確的紀錄，而不是再去算一遍
